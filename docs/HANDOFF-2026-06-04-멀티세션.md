## 4. 공통 주의사항 / 운영 학습 (중요 — 다음 세션 필독)

- **git 병렬 충돌**: 여러 세션이 main에 푸시 → 머지 전 항상 `git fetch origin && git rebase origin/main` 필요. 재시작 후 단일 세션이면 완화.
- **editable install 함정**: 프로젝트가 `pip install -e .`로 main repo의 `src`를 가리킴.
  - `python -c "..."` (cwd=worktree) → **worktree/src 사용** (정상).
  - `python script.py` (스크립트가 job tmp 등) → **main repo/src 사용** (worktree 수정 미반영!). → 테스트 스크립트는 worktree 안에 두고 실행할 것.
- **테스트 실행**: `"C:/Users/af006/stock_report/.venv/Scripts/python.exe" -m pytest tests/ -q` (worktree에서). 88개 통과 기준.
- **GitHub Pages**: push마다 재빌드(~30-60s 일시 불가), 시간당 ~10빌드 소프트제한 → 잦은 재발행 시 "연결끊김"처럼 보임. **불필요한 재발행 자제**.
